# Authelia test environment

Local [Authelia](https://www.authelia.com/) instance for testing OpenID Connect 1.0
authentication flows with the SvelteKit app.

## Quick start

```bash
$ cd authelia-test-env
$ podman compose up -d
$ podman compose logs -f authelia
```

Authelia is now available at **https://127.0.0.1:9091**.

> The server uses a **self-signed certificate** (generated by `openssl`).
> Your browser will show a security warning — accept it and proceed.
> For the SvelteKit app (Node.js), set `NODE_TLS_REJECT_UNAUTHORIZED=0` in `.envrc`
> or when running arctic's OIDC client (dev only).

## Test users

| Username  | Password       | Email                |
| :-------- | :------------- | :------------------- |
| `johndoe` | `testpassword` | john.doe@example.com |

### Adding or changing a password

Generate a new argon2id hash with Authelia's CLI:

```bash
$ podman compose exec authelia authelia hash-password <your-password>
```

Then update `configuration/users.yml` with the output hash and restart:

```bash
$ podman compose restart authelia
```

## OIDC client configuration

The Authelia instance is pre-configured with one OIDC client:

| Parameter     | Value                                       |
| :------------ | :------------------------------------------ |
| Client ID     | `my-app`                                    |
| Client secret | `client-secret-password`                    |
| Redirect URI  | `http://localhost:5173/login/oidc/callback` |
| Scopes        | `openid`, `profile`, `email`                |

## Connecting from the SvelteKit app

Follow [Getting Started](../README.md#getting-started) to start PostgreSQL and the dev server.
No additional user creation is needed — `pnpm add-user` during setup already links `john.doe@example.com` to the OIDC identity.

Then open <http://localhost:5173/login> and click **Sign in with Authelia**.

## OIDC issuer metadata

The OpenID Connect Discovery endpoint is at:

```
https://127.0.0.1:9091/.well-known/openid-configuration
```

## Cleanup

```bash
$ podman compose down -v
```

The `-v` flag removes the named volume (`authelia_data`) containing the SQLite
database and notification history.
